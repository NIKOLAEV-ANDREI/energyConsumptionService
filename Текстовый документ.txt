Цель проекта:
Разработать веб-приложение для анализа и оптимизации домашнего энергопотребления, позволяющее пользователям отслеживать использование электроэнергии по отдельным приборам, рассчитывать ежедневные расходы и получать персонализированные рекомендации по снижению потребления.

Технологический стек:

Клиент: чистый HTML, CSS, JavaScript (без фреймворков)
Сервер: Java Servlets (без Spring, Hibernate и других фреймворков)
База данных: MySQL (через XAMPP)
Сервер приложений: Apache Tomcat 9
Основные функции:

Аутентификация:
Регистрация и вход с валидацией логина и пароля
Хеширование паролей с использованием SHA-256 и соли
Управление сессиями через HttpSession
Анализ энергопотребления:
Ввод данных по приборам: тип, мощность (Вт), время использования (часы/день)
Автоматический расчёт:
Потребление в кВт·ч/день
Стоимость на основе пользовательского тарифа (руб./кВт·ч)
Отображение общей статистики и сравнение с средним потреблением по России (7.5 кВт·ч/день)
Интеллектуальные рекомендации:
Динамические советы на основе:
Типа прибора (холодильник, кондиционер и др.)
Времени использования
Общего уровня потребления
Примеры:
«Ваш кондиционер работает 10 часов в день — это 3 кВт·ч. Попробуйте использовать вентилятор (0.1 кВт·ч).»
«Холодильник — основной потребитель. Убедитесь, что дверца плотно закрыта.» 
Интерфейс:
Минималистичный, адаптивный дизайн
Анимация ошибок при валидации форм
Реальное время пересчёта при изменении параметров
Поддержка сохранения и загрузки данных из БД
Архитектура БД:

Таблица users — учётные записи
Таблица appliances — приборы (мощность, время использования)
Таблица tariffs — индивидуальные тарифы пользователей
Особенности реализации:

Полностью на «чистых» технологиях без сторонних библиотек
Безопасное хранение данных и защита от базовых уязвимостей
Человеко-ориентированный UX с акцентом на энергосбережение
Результат:
Рабочее веб-приложение, помогающее пользователям осознанно управлять энергопотреблением, снижать расходы и вносить вклад в экологическую устойчивость.
